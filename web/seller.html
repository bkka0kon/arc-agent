How it works

3 steps
Step 01

Wrap your endpoint

Put a tiny requirePayment() guard in front of any Express route. It returns 402 with a signed quote until the agent attaches a valid payment header.

Step 02

Set a price

Specify a per-call price in USDC — typically $0.0005–$0.05. The guard verifies + settles through Circle Gateway before your handler runs.

Step 03

List + get paid

Add a row to web/data.js. Agents browsing that category route traffic to you. Revenue accrues to your Gateway balance; withdraw anytime.

Minimal seller server

seller-api/server.js
Express — gate any route with one guard
// seller-api/server.js — speaks Circle Gateway's x402 REST API directly.
import express from "express";
import "dotenv/config";

const SELLER  = process.env.SELLER_WALLET_ADDRESS;   // public 0x… revenue wallet
const GATEWAY = "https://gateway-api-testnet.circle.com";
const USDC    = "0x3600000000000000000000000000000000000000"; // USDC on Arc

// 402 until a valid PAYMENT-SIGNATURE is verified + settled via Gateway.
function requirePayment(price, description) {
  return async (req, res, next) => {
    const sig = req.get("PAYMENT-SIGNATURE");
    if (!sig) return res.status(402).json(quote(req, price, description));
    const ok = await verifyAndSettle(sig);  // POST /v1/x402/verify + /settle
    if (!ok.success) return res.status(402).json({ error: ok.reason });
    req.payment = ok; next();
  };
}

const app = express();
app.get("/v1/price/:token", requirePayment("0.001", "Token price on Arc"), (req, res) => {
  res.json(getTokenPrice(req.params.token)); // ← your real data source
});
app.listen(3000);

Full source incl. verify/settle helpers: seller-api/server.js. When Circle publishes the @x402/core + @x402/evm peers, swap the two REST helpers for the @circle-fin/x402-batching SDK — no route changes.

~0 gas

Platform fee
0%
Arc Agentic charges nothing on top — you receive the full price you set.
