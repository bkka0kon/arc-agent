Long prompt · token analysis end-to-end
You are an autonomous research agent on Arc Network.
Use Arc Agentic to gather data and reason.

Task: Run the "Token Pulse" bundle for token: ${TOKEN}.

Process:
  1. GET  https://data.arc-agentic.dev/v1/price/${TOKEN}
  2. GET  https://data.arc-agentic.dev/v1/liquidity/${TOKEN}-USDC
  3. POST https://data.arc-agentic.dev/v1/sentiment   { token: "${TOKEN}" }
  4. If |change24h| > 5%:
     - POST https://news.arc-agentic.dev/v1/macro     { token: "${TOKEN}" }
     - GET  https://data.arc-agentic.dev/v1/flow/${TOKEN}
  5. Each call returns HTTP 402 first — sign with your Agent Wallet
     (max $0.005 per call) and retry.
  6. Return JSON: summary, price, liquidity, sentiment, optional news + flow.

x402 payment exchange (HTTP level)
# 1. Unpaid request → 402 Payment Required
GET /v1/price/ETH HTTP/1.1
Host: data.arc-agentic.dev

HTTP/1.1 402 Payment Required
PAYMENT-REQUIRED: {
  "accepts": [{
    "scheme": "exact", "price": "$0.001",
    "network": "arc-testnet",
    "payTo": "0xSellerWalletAddress",
    "asset": "0x3600000000000000000000000000000000000000",
    "maxTimeoutSeconds": 300
  }],
  "resource": "https://data.arc-agentic.dev/v1/price/ETH"
}

# 2. Sign with Agent Wallet, retry with PAYMENT-SIGNATURE header
GET /v1/price/ETH HTTP/1.1
PAYMENT-SIGNATURE: {signed-payload}

HTTP/1.1 200 OK
{ "token": "ETH", "price": 3120.45, "change24h": 2.3, ... }
